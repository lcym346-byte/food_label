# Food Label Pro 使用手冊

## 1. 首次開啟

系統會自動建立原料種子資料、示範配方與法規摘要。所有資料保存在本機裝置，不會上傳到伺服器。

## 2. 建立原料

1. 點選「原料庫」。
2. 點選「新增原料」。
3. 若不知道營養數值，先輸入原料名稱，例如「牛奶」、「地瓜粉」、「雞胸肉」。
4. 點選「查詢/選擇營養資料」，系統會優先搜尋內建 TFND/TFDA 離線食品營養成分資料庫（台灣政府 2025 UPDATE1）；即使公開網站查不到，仍可從離線候選項目選擇。
5. 若裝置可連線，系統也會輔助查詢 Open Food Facts 與 USDA FoodData Central 公開資料。
6. 從候選資料點選「帶入」後，請人工複核數值與來源，再按「儲存」。
7. 若沒有合適候選資料，也可手動填入每 100g 的熱量、蛋白質、脂肪、碳水化合物、糖、鈉等資料。
8. 儲存後即可在配方中選用。

## 3. 更新 TFND/TFDA 離線資料庫

系統已內建精簡版台灣政府食品營養成分資料庫，來源包含 `https://consumer.fda.gov.tw/Food/TFND.aspx?nodeID=178` 的 2025 UPDATE1 EXCEL，以及 TFDA OpenData 輔助欄位。若日後要更新資料，可由維護者在專案根目錄執行：

```bash
python3 scripts/update_tfda_nutrition.py
```

腳本會重新讀取 TFND 頁面、下載最新版官方 EXCEL、產生 `web/data/tfda_nutrition_compact.json`，並同步 Android assets。正式商品標示仍應以最新法規、檢驗報告或供應商規格書複核。

## 4. 匯入原料 CSV

CSV 欄位建議使用：

```csv
name,source,calories,protein,fat,saturatedFat,transFat,carbohydrate,sugar,sodium,fiber,tags
```

在「原料庫」點選「匯入 CSV」後選擇檔案即可。

## 5. 建立配方

1. 點選「配方」。
2. 點選「新增配方」。
3. 輸入配方名稱、包裝總重、每份重量與標示格式。
4. 加入每項原料與投入重量。
5. 儲存後即可產生標示。

## 6. 產生營養標示與列印

1. 點選「標示產生」。
2. 從上方下拉選單選擇配方。
3. 在左側「**本批次列印資訊**」卡片填寫：
   - 有效日期模式：製造日 + 保存月數（推薦）或固定有效日期。
   - 製造日期：本批次實際生產日。
   - 保存月數：留空則使用配方預設值。
   - 批號（選填）：例如 `A20260516-01`。
   - 保存條件覆寫（選填）：留空則使用配方預設值。
4. 此區資料只作為「本次列印」用，**不會寫回配方**。重新整理頁面仍會保留，分頁關閉後清空。
5. 確認標籤預覽無誤後，點「**套用並列印**」即可使用瀏覽器或 Android 系統列印服務輸出。
6. 列印時，畫面上的「本批次列印資訊」卡片、最下方稽核提示行不會印出來。

## 7. 瀏覽 TFDA 資料庫

1. 點選主導覽列「**TFDA 資料庫**」分頁。
2. 上方搜尋框可即時過濾食品名稱、英文名或分類。
3. 可依分類下拉、依名稱／熱量／蛋白質等欄位排序。
4. 每筆右側按「**採用為新原料**」會開啟原料對話框並預填數值；確認後按儲存即可。
5. 在原料對話框內按「**瀏覽全部資料庫**」也可進入此頁，選定後會自動填回對話框。




## 8. 備份與還原

- 「設定」→「完整備份 JSON」：下載所有本機資料。
- 「設定」→「還原備份」：選擇先前備份 JSON 還原。
- 「設定」→「重置示範資料」：清除目前資料並恢復初始種子資料。

## 9. Android APK 安裝

1. 從 GitHub Actions 下載 `food-label-pro-apk` artifact。
2. 解壓取得 `app-release.apk`。
3. 傳到 Android 裝置。
4. 允許「安裝未知來源」後安裝。
5. Android 7.1.1 以上版本可使用。
